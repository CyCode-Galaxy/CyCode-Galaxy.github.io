What are Functions?

In computer programming, a function is a block of reusable code that performs a specific task. It's a self-contained unit that can take input, process it, and return an output. Functions help organize code, make it easier to read, and improve overall code structure.

Real-World Analogy

Functions are like recipes - you define the steps once, then can "call" the recipe whenever you need to make that dish, just changing the ingredients (parameters) as needed.

Creating Functions

Here's how to define and call functions in both languages:

JavaScript
// Function definition
function greet(name) {
    return "Hello, " + name + "!";
}

// Arrow function (modern)
const greet = (name) => {
    return `Hello, ${name}!`;
};

// Function call
const message = greet("Alex");
console.log(message); //"Hello, Alex!"
Python
# Function definition
def greet(name):
    return "Hello, " + name + "!"

# Function call
message = greet("Alex")
print(message) # "Hello, Alex!"

Parameters vs Arguments

Parameters

Variables listed in the function definition

// JavaScript
function sum(a, b) {
    // a and b are parameters
    return a + b;
}
Arguments

Actual values passed to the function

// JavaScript
sum(5, 3);
// 5 and 3 are arguments

Return Values

Functions can return values back to the code that called them:

JavaScript
function calculateArea(width, height) {
    const area = width * height;
    return area; // Returns the result
}

const roomArea = calculateArea(10, 12);
console.log(roomArea); // 120
Python
def calculate_area(width, height):
    area = width * height
    return area # Returns the result

room_area = calculate_area(10, 12)
print(room_area) # 120

Default Parameters

You can set default values for parameters:

JavaScript
function createUser(name, role = "student") {
    console.log(`Name: ${name}, Role: ${role}`);
}

createUser("Alex");
// Name: Alex, Role: student
createUser("Sam", "teacher");
// Name: Sam, Role: teacher
Python
def create_user(name, role="student"):
    print(f"Name: {name}, Role: {role}")

create_user("Alex")
# Name: Alex, Role: student
create_user("Sam", "teacher")
# Name: Sam, Role: teacher

Variable Scope

Variables have different visibility depending on where they're declared:

JavaScript
// Global scope 
const globalVar = "I'm global";

function testScope() {
    // Function/local scope
    const localVar = "I'm local";
    console.log(globalVar); // Accessible
}

console.log(localVar); // Error! Not accessible
Python
# Global scope
global_var = "I'm global"

def test_scope():
    # Function/local scope
    local_var = "I'm local"
    print(global_var) # Accessible

print(local_var) # Error! Not accessible
Coding Challenge

Create a function that:

  1. Takes a temperature in Celsius
  2. Converts it to Fahrenheit
  3. Returns the converted value
  4. Formula: F = (C × 9/5) + 32
JavaScript Solution
Python Solution
// JavaScript Solution
function celsiusToFahrenheit(celsius) {
    return (celsius * 9/5) + 32;
}

// Test
console.log(celsiusToFahrenheit(25)); // 77
# Python Solution
def celsius_to_fahrenheit(celsius):
    return (celsius * 9/5) + 32

# Test
print(celsius_to_fahrenheit(25)) # 77

Key Takeaways

  • Functions encapsulate reusable code
  • Parameters are inputs, return values are outputs
  • Variables have scope (global vs local)
  • Default parameters provide fallback values
  • Well-designed functions make code more organized and maintainable
