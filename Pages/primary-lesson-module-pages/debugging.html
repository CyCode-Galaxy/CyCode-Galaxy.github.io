Debugging (Fixing Errors)

20 min

What is Debugging?

In computer programming, debugging is the process of finding and fixing errors or bugs in the source code of any software. Every programmer makes mistakes, and debugging is how we find and correct them!

Fun Fact!

The term "bug" in programming comes from a real insect! In 1947, engineers found a moth stuck in a computer that was causing errors. They "debugged" the computer by removing the moth!

Debugging Tools and Techniques

console.log()

The console.log() method helps us print strings, numbers, javascript objects or variables in the console and you can see what your code is doing:

let number = 5; console.log("The number is", number);
Error Messages

The error message helps us to identify the error in the code. They tell you what went wrong and often where it happened.

// Example error:
// Uncaught TypeError: Cannot read
// property 'length' of undefined
Small Pieces

Small piece technique means instead of testing the whole program, you test one small piece at a time. This helps you isolate where the problem is.

// Test one function at a time
testAddition();
// Then test the next testSubtraction();
Buggy Code Example

This code has several bugs in it. Can you spot them?

function addNumbers(a, b) {
return a + b
}
let result = addNumbers(5, '10')
console.log("The sum is: " result)
Fixed Version
function addNumbers(a,b) {
// Convert strings to numbers
return Number(a) + Number(b);
}
let result = addNumbers(5, '10');
console.log("The sum is: " + result);
